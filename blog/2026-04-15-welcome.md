Welcome to the Verum documentation site.

Verum is a verifiable systems language — refinement types, dependent
types, cubical HoTT, dual-solver SMT verification (Z3 + CVC5), a
three-tier memory model, and explicit context in place of globals.

This blog will cover release notes, design essays, and deep dives into
the language's internals. For the language itself, start with the
[introduction](/docs/intro) or jump straight to the [language tour](/docs/getting-started/tour).

<!-- truncate -->

## What's shipped in 0.32

- **Phase A–D complete**: dependent types, cubical normaliser,
  computational univalence, and proof system are all wired in and
  validated (1506/1507 conformance checks pass).
- **Dual SMT backend**: Z3 + CVC5 with capability-based routing,
  portfolio cross-validation, and per-obligation caching.
- **VBC cubical codegen**: `CubicalExtended` opcode with 17 sub-ops
  closes the proof/execution gap.
- **Full tooling**: CLI (33+ commands), LSP, Playbook TUI, REPL, DAP,
  incremental compilation.
